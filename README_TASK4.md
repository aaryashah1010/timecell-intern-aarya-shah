# Timecell Task 4 - Portfolio Strategy Improvement Simulator

Task 4 extends the CLI risk engine into a strategy simulator:

```text
Portfolio Risk Analysis -> Strategy Detection -> Better Strategy Suggestion
-> Comparison -> Optional AI Verdict
```

The suggested portfolio is always generated by deterministic rules. AI does not
choose allocation numbers; it only explains the comparison after the rule-based
optimizer has finished.

## Run

```bash
python task4_optimizer.py
```

Optional portfolio JSON:

```bash
python task4_optimizer.py --portfolio path/to/portfolio.json
```

Optional AI verdict:

```bash
python task4_optimizer.py --ai
```

You can also start Task 4 through the combined entry point:

```bash
python main.py --optimize
```

## Strategy Modes

The CLI detects the current strategy using Task 1 metrics from
`compute_risk_metrics(portfolio)`.

Aggressive:

- loss is above 40%, or
- runway is below 12 months, or
- concentration warning is true.

Conservative:

- loss is below 20%,
- runway is at least 24 months,
- concentration warning is false.

Balanced:

- anything between those two conditions.

## Target Rules

Conservative:

- max risky asset allocation: 25%
- minimum defensive allocation target: 60%
- redistributes excess to Cash, Bonds, then Gold

Balanced:

- max risky asset allocation: 35%
- minimum defensive allocation target: 40%
- redistributes excess to Bonds, Cash, then Gold

Aggressive:

- max risky asset allocation: 50%
- prevents concentration above 50%
- redistributes excess lightly to Gold, then Cash

Defensive assets are Cash, Bonds, and Gold.

## Name Normalization

Task 4 canonicalizes common aliases so duplicate rows do not appear in the
allocation output.

Examples:

```text
eth, ethereum        -> ETH
btc, bitcoin         -> BTC
nifty, sensex, stock -> Equity
bonds, bond          -> Bonds
cash, fd, savings    -> Cash
gold                 -> Gold
```

So this kind of broken output is avoided:

```text
Bonds 0.0% -> 25.0%
bonds 20.0% -> 0.0%
```

```text

Metric                  Current             Balanced
---------------------------------------------------------------
Post-crash value        INR 60,500          INR 69,000
Loss %                  39.5                31.0
Runway months           1.2                 1.4
Ruin Test               FAIL                FAIL
Largest Risk Asset      ETH                 ETH
Concentration           YES                 NO

ALLOCATION CHANGE:
ETH          50.0% ->   35.0%
Bonds        20.0% ->   25.0%
Cash         30.0% ->   25.0%
Gold          0.0% ->   15.0%
```

Task 4 also prints an impact summary:

```text
======================= IMPACT SUMMARY =======================

Loss reduced       : 39.5% -> 31.0%  (better)
Runway improved    : 1.2 -> 1.4 months  (still low)
Concentration      : YES -> NO  (fixed)
Ruin Test          : FAIL -> FAIL  (still unsafe)

Result: Improved but still unsafe
```

If the suggested portfolio still fails the ruin test, the CLI prints a next
action section explaining that the user may need more defensive allocation,
lower expenses, or more capital.

## AI Verdict

The AI verdict is optional. It receives:

- current metrics
- suggested metrics
- allocation changes
- impact summary

The AI prompt explicitly says:

- do not invent allocation numbers
- do not suggest a new portfolio
- explain only the computed comparison
- mention if the suggested portfolio is improved but still unsafe

If OpenAI is unavailable, the CLI keeps the rule-based output and prints a clean
fallback message.

## Limitations

This simulator uses simple rule-based crash assumptions. It is useful for an
internship technical test and portfolio-risk education, but it is not investment
advice and does not model taxes, liquidity, correlation, investor age, income,
or real-time market conditions.

## Task 4 — Crash Scenario Story Generator

### What This Builds
A CLI tool that stress-tests a portfolio against 5 AI-generated macro
scenarios. ChatGPT generates the story and shock percentages.
Python computes every number. Nothing crosses that boundary.

### Architecture

  ┌──────────────────────────────────────────────────────┐
  │               task4_crash_story.py                   │
  │           (orchestrator — entry point)               │
  └───┬──────────────────┬──────────────────┬───────────┘
      │                  │                  │
      ▼                  ▼                  ▼
  ChatGPT           crash_engine       breakpoint
  scenario_          .py (math)        _detector.py
  generator.py                         (binary search)
      │                  │
      │  shock_map        │  imports
      │  (% values only)  ▼
      └─────────────▶ risk_calculator.py
                       (Task 1 engine)
                          │
                          ▼
                    report_formatter.py
                    (terminal output)

### Core Principle
ChatGPT generates scenarios and shock percentages only.
Python computes all INR values, runway, and verdicts.
This directly reflects TimeCell's principle:
"Math computed in code, not guessed by a language model."

### Why ChatGPT's Shock Values (Not crash_assumptions.py)
crash_assumptions.py contains generic historical averages.
ChatGPT provides scenario-aware values:
  Generic map: BTC = -70% always
  ChatGPT:     BTC = -71% (crypto crackdown)
               BTC = -20% (RBI rate hike — barely affected)
               BTC = -55% (global credit freeze)
The prompt forces ChatGPT to cover every asset, so no fallback is needed.

### How To Run
  python task4_crash_story.py              # default portfolio
  python task4_crash_story.py --input      # enter your own portfolio
  python task4_crash_story.py --dry-run    # test math, no API cost
